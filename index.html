Diagrama de Cascada

El desarrollo del proyecto Arkanoid se estructura bajo el modelo en cascada, caracterizado por una secuencia lineal de fases, donde la finalización de una etapa da paso a la siguiente. Esta organización permite una planificación rigurosa y un control preciso del avance. El proceso inicia con una reunión inicial, cuyo propósito es establecer los objetivos generales y evaluar la viabilidad técnica y estratégica del proyecto. Esta instancia resulta fundamental para alinear expectativas, delimitar el alcance y definir una visión compartida por el equipo líder del proyecto. Seguidamente, se realiza una reunión legal, orientada a revisar aspectos contractuales, licencias y derechos de propiedad intelectual. Esta fase temprana es clave para prevenir conflictos jurídicos que podrían surgir durante el desarrollo o la posterior distribución del juego.

Una vez definidos los aspectos legales, se lleva a cabo el onboarding del equipo, donde se asignan herramientas, accesos y responsabilidades. Esta etapa asegura que todos los integrantes comprendan la estructura del proyecto y estén técnicamente preparados para participar. Posteriormente, se desarrollan reuniones por área, con el objetivo de levantar requerimientos específicos según el rol de cada integrante. Esta actividad permite construir una visión detallada del producto desde distintas perspectivas, enriqueciendo la calidad de los requerimientos.

Con la información recopilada de las fases anteriores, se elabora la documentación de requerimientos, donde se formalizan las necesidades funcionales y no funcionales del sistema. Esta documentación constituye la base técnica sobre la que se construirá el producto. En este punto, al contar ya con una visión clara y estructurada del proyecto, es oportuno llevar a cabo la estimación de costos. Esta actividad permite evaluar la viabilidad financiera del desarrollo, optimizar la asignación de recursos y asegurar que el proyecto se mantenga dentro de los márgenes presupuestarios establecidos.

Después, se ejecuta la fase de branding, que define la identidad visual del juego, incluyendo logotipo, paleta de colores y elementos gráficos clave. Esta coherencia estética será determinante en la percepción del usuario final. Concluida esta etapa, el proyecto está listo para dar inicio a la fase técnica con la creación del repositorio y los entornos de desarrollo, asegurando que los flujos de trabajo y control de versiones estén disponibles para todos los programadores.

A continuación, se realiza el diseño del juego, donde se definen las mecánicas, niveles y elementos de jugabilidad. Posteriormente, se aborda el diseño visual, donde se crean personajes, fondos, efectos, elementos gráficos y componentes de interfaz. Estos diseños son esenciales para orientar el desarrollo técnico posterior y se complementan con la creación de mockups funcionales y el diseño de front, que permiten representar visualmente la estructura del juego antes de su implementación.

El siguiente paso consiste en el diseño de la base de datos, donde se define cómo se almacenará, organizará y gestionará la información dentro del sistema. Para este proyecto, se ha optado por un modelo de base de datos relacional, el cual permite estructurar los datos mediante tablas con relaciones definidas entre ellas. Este enfoque facilita la integridad, consistencia y eficiencia en el acceso a la información, y establece la lógica de persistencia que será utilizada por los componentes del backend durante la ejecución del juego.

Con todos los elementos de diseño definidos, se inicia el desarrollo en Unity, donde se implementa la lógica del juego utilizando el motor gráfico. A continuación, se lleva a cabo el desarrollo backend, encargado de construir los servicios API, gestionar la lógica de negocio y establecer la conexión con la base de datos. Se desarrolla el frontend, que implementa la interfaz de usuario y se comunica con el backend para ofrecer una experiencia fluida y funcional.

Una vez que las funcionalidades principales del sistema están implementadas, se procede a incorporar el diseño de sonido, el cual incluye la composición de música y la creación de efectos auditivos. Esta etapa se realiza en este punto del proyecto para asegurar que los sonidos se integren de manera contextual con las mecánicas y eventos del juego ya desarrollados, enriqueciendo así la experiencia inmersiva del jugador y mejorando la retroalimentación sensorial durante la interacción con el sistema.

Una vez completado el desarrollo, se llevan a cabo pruebas en desarrollo, verificando la funcionalidad en un entorno controlado. Luego, se realiza el despliegue en producción y se efectúan pruebas en el entorno real, garantizando la estabilidad del sistema. Finalmente, se realiza una revisión y aprobación por parte del cliente, y si todo está conforme, se da inicio a las actividades de ventas y distribución.


Diagrama de Gantt

En la primera iteración se sentarán las bases generales del proyecto, se abordarán reuniones clave, se llevará a cabo el levantamiento de requerimientos críticos y la configuración técnica inicial. También se avanzará en el diseño general, tanto visual como funcional, y se desarrollará una primera versión jugable (demo) centrada en probar la mecánica principal del juego, sin abordar todavía la implementación de elementos estéticos.

Luego de que la mecánica básica del juego sea alcanzada, la segunda iteración se enfocará en el desarrollo de nuevas funcionalidades y la implementación de algunos elementos estéticos y sensoriales. Se definirán requerimientos adicionales en nuevas reuniones por áreas, y se avanzará en el desarrollo de niveles, power-ups y efectos de sonido. El objetivo será obtener un demo funcional más avanzado que permita evaluar la integración de estos elementos y mejorar la experiencia de juego.

En la tercera iteración se trabajará en mejorar la estética general (mejoras visuales, mejoras de sonido) e incorporar funcionalidades clave como la puntuación y la persistencia de datos (con el objetivo de que los jugadores puedan ver su progreso). Se levantarán nuevos requerimientos, se diseñará la base de datos y se avanzará en el desarrollo backend, además de continuar avanzando en aspectos relativos al branding.

Tras todos los avances previos, se espera que para la cuarta iteración se añada la funcionalidad de login y se completen los niveles del juego en su totalidad, lo que permitirá tener un prototipo funcional completo del juego. Se efectuarán pruebas en desarrollo y en producción, y, además, se evaluará el flujo completo de la experiencia del usuario, desde el ingreso al sistema hasta la navegación por los niveles, para determinar cambios o mejoras necesarias.

La quinta iteración se centrará en efectuar los cambios y aplicar las mejoras funcionales o de flujo que se hayan visto necesarias tras la cuarta iteración, a fin de preparar el juego para su lanzamiento oficial. Se ultimarán aspectos legales, se efectuarán los últimos cambios del diseño sonoro, se optimizará el rendimiento y se gestionará todo lo necesario para la venta y distribución del juego en las plataformas correspondientes.

Para concluir, consideramos que de esta manera se abordan todas las necesidades del proyecto de forma estructurada y progresiva, permitiendo una implementación continua y eficiente de funcionalidades y mejoras.
